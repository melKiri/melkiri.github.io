The real problem

The work we wanted
but couldn't justify.

Every design system accumulates a backlog of work that everyone knows needs doing — but the time required to do it properly can never be justified against the immediate roadmap. So it gets deferred. Workarounds get built instead. The debt grows quietly.

At Pushpay, this meant components that shipped without documentation. Variable structures that needed auditing across 17,000 tokens. A library that needed cross-referencing against Storybook and GitHub to find inconsistencies. Work that would take weeks of focused effort from someone who had neither the weeks nor the focus to spare.

"It was quicker to find workarounds than to fix it properly. The time it would take couldn't be justified. So the debt stayed."

AI didn't make this work faster. It made it possible for the first time. The calculation changed — not because hours got shorter, but because hours got replaced entirely for the parts of the work that were never design decisions in the first place.

The workflow

One workflow.
Not tool-switching.

The key wasn't any single tool. It was connecting them — using Claude as the connective tissue across the entire DS workflow so context doesn't get lost between tools and outputs feed directly into the next step.

The stack
Claude
Reasoning, auditing, generating documentation, creating Jira tickets from findings, making bulk Figma updates
Figma Console MCP
Direct Figma access — read and write variables, audit component usage, push updates without manual work in the UI
DS Assistant MCP

Engineering source of truth — cross-referenced against Figma to find parity gaps and inconsistencies
Atlassian / Jira
Audit findings converted directly into tickets — no manual transcription step
Dovetail · Pendo
Research and usage data feeding design decisions — not just design intuition

What changed

Work that didn't exist
before this workflow.

Some of this work got faster. But the more important category is work that simply wasn't happening — because the cost in time and capacity could never be justified against other priorities.

Full DS audit — Prism + component library

A proper audit of scope, descriptions, naming, variable usage, and component health across 500+ variables and components. The scale made it impractical to do manually — it would surface issues faster than they could be resolved. Now it runs on demand, and findings go directly into Jira as actionable tickets.

Previously: impractical at this scale

Documenting components that shipped without docs

Components get built under deadline pressure, shipped, and never documented. Retroactively writing structured documentation for every component — usage rules, variants, accessibility notes — was a backlog that kept growing. 60 minutes per component became 5 minutes. The backlog is now clearable.

Previously: always deferred, never done

17k variable migration in one day

Updating the entire variable structure — auditing old token usage, replacing hardcoded values, pushing the new semantic architecture across the library. Not 17,000 manual updates. One structured operation, validated, run through Claude directly to Figma. A week of work became a day.

Previously: would have taken weeks

Cross-channel parity checks

Comparing what exists in Figma against what's in Storybook against what's in the GitHub repo — finding where they diverge, where components don't match, where the design intent isn't reflected in the built output. This is the kind of audit that might happen once a year if you're lucky. Now it's on-demand.

Previously: happened rarely, if ever

Icon library rebuild at scale

100 icons, ~30 seconds each to manually find, configure DS specs, and place correctly. That's ~80 minutes of work that required zero design judgment — just pattern recognition and repetition. With Claude, it's a few minutes. The designer focuses on which icons, not on the placing of them.

Previously: 80 minutes of non-design work

The philosophy

Strategic use.
Not blind reliance.

The approach isn't to use AI for everything — it's to use it precisely where human judgment isn't the bottleneck. Test it. Validate it. Run it once a human is no longer needed in the loop.

The design decisions still require a designer. What components exist, how they should behave, what the token architecture should mean — these aren't questions AI answers. What AI replaces is the execution overhead: the migration, the documentation formatting, the audit transcription, the repetitive find-and-replace work that fills hours without requiring thought.

01

Test before trusting

Every workflow gets validated before it runs at scale. A wrong assumption in an automated audit is worse than no audit — it creates false confidence. The human stays in the loop until the output is reliable.

02

Replace overhead, not judgment

AI handles the parts of the work that were never design work to begin with. The decision of what a component should do is still a designer's decision. The migration of that decision across 17,000 tokens is not.

03

Connected, not switched

The value isn't in any single tool — it's in keeping context alive across tools. Figma to Storybook to GitHub to Jira without losing the thread. One workflow, not five separate ones.

04

Make the impossible routine

The best outcome isn't doing existing work faster. It's changing which work is possible — turning the audit that happened once a year into something that runs on demand, and the documentation backlog that was never cleared into something clearable.
